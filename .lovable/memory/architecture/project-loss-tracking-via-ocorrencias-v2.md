# Memory: architecture/project-loss-tracking-via-ocorrencias-v2
Updated: 2026-03-18

As ocorrências operacionais (aba 'Incidentes') são o mecanismo de governança para perdas financeiras. (1) Fluxo: Rastreiam 'valor_risco' ( exposure total da conta via botão 'Usar saldo atual', nunca pré-preenchido) e 'valor_perda'. (2) Auditoria: Datas de início e fim no formato 'yyyy-MM-dd' são processadas no fuso local (SP) para cálculo de MTTR preciso. (3) Liquidação: Ao marcar como 'Resolvido', o sistema registra perdas parciais ou totais no ledger, reduzindo o saldo da bookmaker e impactando o lucro líquido do projeto (LUCRO = APOSTAS + CASHBACK + GIROS - PERDAS). (4) Link confiável: A tabela 'projeto_perdas' possui 'ocorrencia_id' para vinculação direta (substituindo match por descrição). (5) Proteção pós-desvinculação: Se a bookmaker foi desvinculada, a perda é registrada apenas em 'projeto_perdas' sem debitar saldo (já saiu via SAQUE_VIRTUAL). Reabertura e cancelamento respeitam esse estado.
